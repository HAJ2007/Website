HANG-MAN

Github link HangmanGame

This is a digital version of a childhood game.
The game requires the player to guess all the letters of a word when only the length is given.
The player get aroud 5-7 chances to make an incorrect guess depending on how the "hang man" is drawn.
For my game I gave the user 5 turns to guess the incorrect letters
If the player made a wrong guess after the 5th chance for wrong guesses are given the user will loose.

CORE FEATURES:

  • Validation: All inputs are checked against a preset criteria to prevent the user inputs causing an erro
    This is especially because the user inputs otherwise can be unpredictable.

  • GUI: A simple interface is implemeted using Tkinder library. This makes the application user friendly.

  • Multiple windows: Rather than containing all the processing in one window I have used specific windows for specific tasks

       Start Window:Contains The title of the Game and the start button to begin the game

       Gameplay window:Additional to the features that displays the game functions
       there is also a Menu button which has a pop up section.
       This popup displays buttons to go back to the start(HOME button) and 'REPLAY' button.
       This improves accessibility.
       When a game is won or lost the guess button and Enty smace is blocked to prevent userinputs

       EndScreen: When the game is lost or won a button to end game appears on 'gameplay'window.
       This button allows to end the game and go to the next window containing the buttons in menu
       When improving the current version, this window promises easy integration of new features
       such as score.

  • Hang- Man: Different sections of hangman is added based on the number of wrong guesses.
    The drawing pervorms similar fuction to that of a life metre peresent is ames ususally.
    This helps the user keep trach of the chances the user have left.

ENCODE and DECODE

Github link Encode & Decode

The program consists of a simple encoder and decoded based of Caesar cipher
The user is able to chose between Encoding and Decoding and input.
The decoding uses brute force attack where all the 25 possible shifts are shown to the user.
CORE FEATURES:

  • Encoding: The user can input a string that is greater 1 to 45 characters in legth(including spaces).
    The user can then either selct to randomly generate a shif or input a shift number to encode the input
    The shift number need to be less than 25 and greater than 0. There are instruction for guiding the user
    If the random variable is not clicked and a shift is not entered the progtam will show 2 pop ups showing error.The user simply eed to click ok to get result

  • Decoding: Because all the inputs have to be shown due to the brute force nature of the program
    The character length for input is between 1 and 45.
    Instructions to guide the user are given. After clicking ok all outputs are given

  • Menu-Button: There is a menu button on both encode and decode windows allowing one to return only to home. The menu button is toggled using a variable that stores a value when the menu button function is called. The pop up is opened nd colsed based on this variabel.

  • GUI: The simple interface using tkinder allows the user to see the result within a clean layout
    The colour scheme is composed of bright blue and orange, which is cmplemetary colours providing a vibrant layout. The encoding and decoding have a mixture of colours.

  • Case and Punctuation: All the punctuation is ignored when shifting, so output will retain the Punctuation
    The shift is sensitive to case, so if the input is in upper case, the shift will be in upper case for both decode and encode

  • Validation: The validation is the same for encoding and decoding in terms of input characters. Other validation regarding shift and randomisation is specific to encoding

DEDUCTION

Github link Deduction

This is a simple command-line game that works my conditional statemtments, random librabry and list data type. The game asks the player to guess a four-digit number that is randomly selected. Hints are given to the user to help deduce the number. These hints depend on the individual input. There is no clear loss criteria but to win, you have to guess all numbers correctly.
CORE FEATURES:

  • Clues: The clues are given based on input, then randomly shuffled to increase the difficulty.
       HOT: this means one of the guessed numbers is correct and in the right place
       WARM: this means one of the guessed numbers is correct, but in the wrong place
       COLD: The number is incorrect
    Because the clues are shuffled, the input position is not matched to the clues. This is done especially because the initial generation of the clue is based on position of the digint

  • Validation: The inputs are checked against 2 criteria. Are all inputs consisting of only numbers? Are all inputs 4 digits long?
